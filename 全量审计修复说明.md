## 2. 本轮实际修复

### 2.1 描述文件误判签名证书

Apple / AltSign 返回的证书 Serial 可能省略前导 `0`，而 `embedded.mobileprovision` 中的 X.509 证书 Serial 会保留前导 `0`。旧逻辑按字符串比较，导致同一证书被误报为不匹配。

已新增统一的 `CertificateSerial` 十六进制整数标准化逻辑，并应用于：

- 描述文件证书校验
- 本机 P12 与远端证书匹配
- 证书选择策略
- Apple Portal 证书列表
- 设置页证书状态
- 签名与续签协调器

同时拒绝把 `Seal-D94193D9` 之类的展示名称误当成真实 Serial。

### 2.2 证书满额后无法自动继续签名

旧实现明确拒绝自动撤销证书，与最终规则冲突。

现已改为：

1. 创建证书；
2. Apple 返回容量已满时重新拉取证书；
3. 找出本机没有对应 P12 私钥的远端开发证书；
4. 逐个撤销并立即重试创建；
5. 创建成功后立即刷新并继续当前签名 / 续签；
6. 全部候选处理后仍无名额才返回明确错误。

不会要求用户重启 Seal、清后台或重新点击签名。

### 2.3 Seal 自替换安装可能在真正安装前被系统挂起

旧流程先调用返回主屏幕，再继续执行安装；调用私有 `suspend` 后当前进程可能立即冻结，导致安装调用无法可靠完成。

现已：

- IPA 暂存完成后申请 iOS 后台任务；
- 自动返回主屏幕；
- 在后台任务保护下执行 Minimuxer 安装；
- 成功、失败和取消路径均结束后台任务。

### 2.4 每日自动续签可能把“已签名”误当成“已安装”

旧逻辑可能依据 `lastSignedAt`、`lastInstalledAt` 或数据库中的新描述文件到期时间，将 Seal 自续签标记为成功；这些数据在 iOS 真正替换 Seal 前就可能写入。

现仅接受当前重新启动的 Seal 进程从自身 `embedded.mobileprovision` 读取到的新到期时间作为跨启动成功证据。

- 有旧到期时间：新嵌入到期时间必须真实推进；
- 无旧到期时间：当前嵌入到期时间必须与本次签名描述文件到期时间一致，并晚于续签开始时间；
- 同一天重新启动任务时会更新本次基线，避免残留状态误判。

### 2.5 Seal 自续签身份校验原来是空实现

旧 `SelfRenewalContextValidator` 无条件通过。

现强制校验：

- 当前运行 Seal 的 Bundle ID 与续签目标完全一致；
- 必须存在上次签名绑定的 Apple ID；
- 续签必须使用原 Apple ID；
- 当前嵌入描述文件 Team 可读取时，必须与原账号 Team 一致。

### 2.6 配对文件重新导入后可能继承旧验证状态

旧流程写入新配对文件后尝试删除旧元数据；若元数据删除失败，新文件可能错误显示为已配对。

现会先原子写入“未验证”元数据，再替换配对文件。刚导入固定显示“已导入，待验证”，真实安装通道验证后才升级状态。

### 2.7 刷新队列重复记录可能造成崩溃或重复续签

旧数据损坏或历史版本残留重复 App ID 时，队列字典构建存在重复键风险，并可能重复执行同一 App。

现会：

- 按 App ID 归一化持久化队列；
- 已完成状态优先保留，否则保留最新记录；
- 出队时再次去重；
- 避免同一 App 在一轮队列中执行两次。

### 2.8 IPA 导入记录大小写不一致会生成重复 App

Bundle ID 本质上按不区分大小写匹配。旧导入流程和 Core Data 查询存在大小写敏感路径。

现导入工作流与 Core Data 均使用不区分大小写匹配，并保留同一 App 的稳定记录 ID。

### 2.9 设置页可能把缺失私钥的证书显示为“本机可用”

现只有 Keychain 中 P12 可解析，且解析出的 Serial 与选择证书一致时才显示“本机可用”。

- 没有有效本机证书：显示“签名时创建”；
- 选择记录与本机 P12 不匹配：显示“私钥不可用”。

### 2.10 签名失败页重复出现两个“重试”按钮

当错误主操作本身已经是“重试”时，现不再额外生成第二个重试按钮。

### 2.11 动态 Anisette WebSocket 地址存在强制解包崩溃风险

现对 `URLComponents` 和生成的 WebSocket URL 做显式校验，异常服务器地址返回可处理错误，不再强制解包崩溃。

### 2.12 LocalDevVPN 探测端口存在强制解包

现对 `NWEndpoint.Port` 构造结果做安全校验，非法端口直接返回探测失败。

### 2.13 签名历史统计存在可移除的强制解包

现改为显式 Optional 绑定，避免历史数据异常时因强制解包导致页面崩溃。
